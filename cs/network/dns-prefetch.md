DNS 预解析的 **触发方式** 分两类（两者 **可以叠加** ，效果更好）：

1. **浏览器默认行为**
2. **前端开发者主动优化**

<br/>

**浏览器自动 DNS 预解析（默认开启，不用管）**
现代浏览器（Chrome、Edge、Firefox、Safari）都有 **自动预解析功能** ，会在 **页面加载的空闲阶段**（比如核心内容加载完、浏览器没其他渲染 / 请求工作时），**自动扫描页面中的所有跨域域名**（比如页面是`a.com`，里面有b.com的图片、c.com的链接），并对这些域名做预解析。

**扫描范围**：页面中的`<a>`链接、`<img>`图片、`<script>`脚本、`<link>`样式、AJAX 接口等所有带域名的资源；

**触发时机**：**浏览器空闲时**，不会抢占页面核心加载的资源，不会拖慢页面本身的加载速度。

**前端代码强制 DNS 预解析（开发者主动优化，针对隐藏的域名）**
<u>浏览器自动预解析只能扫页面中可见的域名，但有些域名是隐藏的</u>（比如点击按钮后才会发起的接口、弹窗里的图片），浏览器扫不到，就没法自动预解析。
这时前端开发者会通过一行简单的代码，强制浏览器对指定域名做预解析，保证这些隐藏域名也能提前查好 IP(`<link rel="dns-prefetch" href="域名">`)
