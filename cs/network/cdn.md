CDN（Content Delivery Network）即 **内容分发网络**，是构建在现有网络基础上的分布式网络，核心是 **通过在全球部署边缘节点**，**让用户从就近节点获取内容**，从而 **降低延迟** 、**减轻源站压力**、**提升访问稳定性与安全性**

#### 一、核心定义与本质
本质：不是单台服务器，而是由边缘节点集群、负载均衡系统、内容缓存系统组成的分布式网络，核心逻辑是 “就近取材”，解决跨地域传输的距离与带宽难题。
通俗比喻：源服务器是 “总仓库”，CDN 边缘节点是各地 “社区便利店”，用户无需到总仓库，就近的便利店即可满足需求，速度与稳定性大幅提升。

#### 二、工作流程

用户输入域名访问，本地 DNS 解析请求被导向 CDN 的 GSLB。
GSLB 根据用户位置、节点健康度、负载等选择最优边缘节点，返回节点 IP。
用户向边缘节点发起请求，节点若缓存目标内容则直接返回；若无则向源站请求并缓存后返回。
源站同步内容至各边缘节点，确保缓存一致性。

#### 三、核心价值与应用场景

##### 1. 核心价值
   
+ **降低延迟**：缩短物理传输距离，减少加载时间（如视频秒开、网页速加载）。
+ **减轻源站压力**：分担高并发流量，避免源站因带宽或负载过高宕机。
+ **提升可用性**：多节点冗余，单点故障不影响整体服务，抗 DDoS 能力更强。
+ **降低带宽成本**：减少跨地域长距离传输的带宽消耗，优化成本结构。

##### 2. 典型应用场景
- **静态资源加速**：网站图片、JS/CSS、字体、静态页面。
- **视频 / 直播加速**：长视频、短视频、直播流（如 Netflix、抖音）。
- **下载加速**：游戏安装包、APP 更新包、软件分发。
- **动态内容加速**：API 响应、动态页面（结合边缘计算优化）。
- **安全防护**：DDoS 防护、WAF、SSL/TLS 加速。
 
#### 四、常见误区与补充说明
误区 1：CDN 只能加速静态内容。实际可通过动态路由、边缘计算等技术加速动态内容，如 API 与个性化页面。

误区 2：CDN 可替代源站。CDN 是源站的 “加速与保护层”，无法替代源站的内容存储与业务逻辑处理。

误区 3：CDN 节点越多越好。节点质量（带宽、网络、位置）与调度策略更关键，需平衡覆盖与成本。
