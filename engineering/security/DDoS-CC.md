DDoS 是 **分布式拒绝服务攻击**（Distributed Denial of Service）的缩写，是 **网络中最常见的恶意攻击之一**，核心目的就是 **让你的服务器 / 网站彻底瘫痪**，正常用户无法访问

#### 一、简单说就是「把服务器挤爆」
你可以把自己的服务器想象成一家小超市，正常情况下，超市的收银台、过道能容纳有限的顾客（正常用户请求），大家都能顺利购物。
而 DDoS 攻击，就是攻击者操控成千上万台被病毒控制的电脑 / 设备（叫 “肉鸡”），同时假装成顾客冲向超市，这些 “假顾客” 不买东西，只是占着过道、围着收银台，让真正的顾客挤不进去，超市彻底没法正常营业 —— **服务器的带宽、CPU、内存被海量虚假请求占满，处理不了正常用户的请求，最终卡死、崩溃**

#### 二、为什么是「分布式（Distributed）」？
早期的攻击是单台电脑发起的（叫 DoS 攻击），服务器很容易屏蔽这台电脑的 IP 就搞定了。
而 **DDoS 是分布式** 的，攻击者控制的是一个由成百上千、甚至上万台设备组成的「僵尸网络」，这些设备分布在全球各地，IP 地址五花八门，服务器根本 **没法一次性屏蔽所有攻击源**，防御难度直接拉满。

#### 三、常见的 DDoS 攻击类型

主要针对服务器的两个核心资源，这两种最典型：

+ **带宽耗尽型**：海量虚假请求占满服务器的网络带宽，就像水管被泥沙堵死，正常数据传不进来也出不去；
+ **资源耗尽型**：发起大量需要服务器消耗 CPU / 内存处理的请求，比如让服务器反复做复杂计算，把服务器的硬件资源榨干，直接卡死。
  
#### 四、CDN 是防御 DDoS 的重要手段

**配置 CDN 不仅能提速，还能保护服务器**

- **第一层拦截**：所有用户（包括攻击的虚假请求）都会先访问 CDN 节点，而不是直接访问你的源站服务器，**CDN 会先过滤掉大部分明显的虚假请求**
- **分流扛压**：CDN 是分布式的节点集群，单节点被攻击，其他节点能顶上去，而且 CDN 服务商有超大的带宽和专业的防御设备，**能扛住海量请求，替你的源站挡住绝大部分攻击压力**
- **源站隐藏**：配置 CDN 后，正常用户和攻击者都 **看不到你源站的真实 IP**，攻击者连攻击目标都找不到，自然没法直接攻击源站。
  
#### 五、和 DDoS 容易搞混的「CC 攻击」

CC 攻击是 **DDoS 的一种特殊形式**，专门 **针对动态网页 / 接口**（比如你的服务器有/api/getData接口），攻击者用僵尸网络 **发起大量针对接口的合法请求**（看起来和正常用户请求一样），**让服务器反复处理这些请求，最终耗尽 CPU / 内存，比普通 DDoS 更难防御**，而优质的 CDN 也能通过频率限制、智能识别拦截大部分 CC 攻击。
