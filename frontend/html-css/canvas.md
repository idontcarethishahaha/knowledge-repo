## Cavas - 画布

### 一、画布 ≠ `<canvas>`标签
你在 HTML 里写的 **`<canvas>`标签是页面上的 DOM 元素**，浏览器会像处理 div 一样给它做布局、定位置；

而 **画布是`<canvas>`标签背后的像素容器，是浏览器在内存中开辟的一块矩形像素区域**，<u>标签只是画布的 “展示窗口”</u>

- 标签负责在页面上占位置、定大小（由 CSS 控制显示尺寸）；
- 画布负责真正承载绘制的像素内容（由 JS 设置canvas.width/height定实际像素尺寸）。

### 二、画布的核心特性

1. **画布是像素级的，只有像素矩阵，无任何结构**
画布的底层就是一行行、一列列的像素点组成的矩阵，比如设置canvas.width=300, height=200，就是创建了一块300 列 ×200 行 = 60000 个像素点的画布，每个像素点可以设置颜色、透明度。
你画的矩形、圆形，本质都是给画布中指定范围的像素点设置颜色，画布不会记录 “我画了一个矩形”，只会记录 “哪些像素是红色、哪些是蓝色”。
1. **画布的内容只能通过 JS 绘制 API 操作，无 CSS / 原生属性可改**
画布本身没有任何样式、属性可以通过 CSS 修改，也没有 DOM 节点那样的style、className等属性。
想给画布加内容、改内容，只能通过getContext('2d')获取绘制上下文（相当于 “画笔”），再调用fillRect()、arc()、drawImage()等 API，给像素点赋值 —— 这就是你之前说的 “手动触发绘制”。
1. **画布的修改无局部编辑能力，只能重绘 / 叠加像素**
因为画布只记录像素，不记录绘制的图形结构，所以想修改已画的内容，没有 “选中图形改颜色” 的操作，只能两种方式：
清空画布（clearRect()）后重新绘制所有内容；
在原有像素上叠加绘制新像素（比如在红色矩形上画蓝色圆形，只是让重叠区域的像素变成蓝色）。
唯一的 “局部像素修改”，是通过getImageData()读取指定区域的像素数据，修改后再用putImageData()写回去，相当于手动改像素点颜色。
1. **画布的尺寸<u>有两个维度，必须匹配</u>否则会拉伸**
这是实际开发中最容易踩的坑，**画布的尺寸分像素尺寸（canvas.width/height）和显示尺寸（CSS 的width/height）**
- 像素尺寸：决定画布有多少个像素点，是绘制的 “实际尺寸”；
- 显示尺寸：决定`<canvas>`标签在页面上的展示大小，是浏览器布局的 “视觉尺寸”。
如果两者不匹配（比如像素尺寸 300×200，CSS 显示尺寸 600×400），浏览器会把 60000 个像素点拉伸到 600×400 的区域，绘制的内容就会模糊、变形。
