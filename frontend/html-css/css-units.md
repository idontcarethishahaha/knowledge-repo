## 单位

### 一、绝对单位（固定尺寸，不随环境变化）
绝对单位的 **尺寸固定**，不会因为屏幕大小、父元素或字体大小而改变，适合需要 **精确控制尺寸** 的场景（如打印样式、固定尺寸的 UI 组件）。

|单位|	含义|	换算关系	|示例 & 适用场景|
|:--:|:--:|:--:|:--:|
|px|	像素（Pixel）|	1px = 屏幕最小显示单元|	width: 200px;（最常用，适配屏幕显示）|
|pt|	点（Point）	|1pt = 1/72 英寸|	font-size: 12pt;（打印样式、PDF 文档）|
|in|	英寸（Inch）|	1in = 96px = 72pt|	width: 5in;（打印、物理尺寸标注）|
|cm|	厘米（Centimeter）|	1cm ≈ 37.8px	|margin: 2cm;（打印、物理尺寸）|
|mm|	毫米（Millimeter）|	1mm ≈ 3.78px|	padding: 5mm;（打印、物理尺寸）|
|pc|	皮卡（Pica）|	1pc = 12pt = 1/6 英寸|	极少使用，仅传统印刷领域偶尔用到|

<br/>

### 二、相对单位（动态尺寸，随参考基准变化）
相对单位是开发 **响应式页面的核心**，**尺寸会根据父元素、根元素、视口等基准动态调整，适配不同屏幕大小**

##### 1. 基于字体大小的相对单位

|单位	|参考基准|	示例 & 适用场景|
|:--:|:--:|:--:|
|em	|父元素的字体大小（font-size）|	font-size: 1.2em;（继承父元素字体，适合组件内排版）|
|rem|	根元素（html）的字体大小|	font-size: 1.1rem;（全局统一基准，推荐响应式字体）|
|ex	|小写字母 x 的高度（极少用）|	几乎不用，兼容性差|
|ch	|数字 0 的宽度（极少用）	|适合等宽字体排版（如代码块）|

```css
/* 根元素设置基准字体（rem的参考） */
html {
  font-size: 16px; /* 浏览器默认值，可根据屏幕调整 */
}

/* rem示例：全局字体，适配所有屏幕 */
h1 {
  font-size: 2rem; /* 2*16=32px */
}
p {
  font-size: 1rem; /* 16px */
}

/* em示例：组件内相对排版 */
.card {
  font-size: 14px; /* 父元素基准 */
}
.card-title {
  font-size: 1.2em; /* 1.2*14=16.8px */
  margin-bottom: 0.8em; /* 0.8*14=11.2px */
}
```

<br/>

##### 2. 基于视口（Viewport）的相对单位

**视口指浏览器的可视区域**，这类单位直接适配屏幕大小，是响应式布局的核心。

|单位|	参考基准|	示例 & 适用场景|
|:--:|:--:|:--:|
|vw|	视口宽度的 1%|	width: 50vw;（元素宽度占屏幕一半）|
|vh|	视口高度的 1%|	height: 100vh;（元素占满屏幕高度）|
|vmin|	vw 和 vh 中较小的那个 1%|	width: 80vmin;（适配横竖屏，保证元素不溢出）|
|vmax|	vw 和 vh 中较大的那个 1%|	font-size: 5vmax;（横竖屏都保证字体大小）|

<br/>

##### 3. 基于百分比（%）

百分比是最基础的相对单位，**参考基准是父元素的对应属性**（如宽度 % 参考父元素宽度，高度 % 参考父元素高度）。

<br/>

##### 4. CSS3 新增单位

|单位|	含义|	适用场景|
|:--:|:--:|:--:|
|fr|	网格布局的份数|	Grid 布局中分配剩余空间（如grid-template-columns: 1fr 2fr;）|
|lh|	行高（line-height）|	基于行高的间距（margin-bottom: 1lh;）|
|dvh/dvw|	动态视口单位|	适配移动端地址栏 / 导航栏变化的视口（解决 vh 在移动端的兼容问题）|
