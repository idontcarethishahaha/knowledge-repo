## 闭包

### 一、 定义

闭包是 **内层函数** 即使脱离了定义它的 **外层函数作用域** （比如在全局执行），依然能访问外层函数中的变量 —— 相当于 **内层函数携带着外层的变量环境**

### 二、形成闭包的核心条件
- 存在函数嵌套（内层函数 + 外层函数）；
- 内层函数引用了外层函数的局部变量；
- 内层函数被暴露到外层函数作用域之外执行

### 三、 闭包的典型场景

##### 1. 模拟私有变量
```js
function createCounter() {
  let count = 0; // 私有变量，外部无法直接访问
  return {
    add: () => count++,
    getCount: () => count
  };
}
const counter = createCounter();
counter.add(); // 1（只能通过指定方法修改）
console.log(counter.count); // undefined（外部无法直接访问）
```

##### 2. 解决循环变量共享问题

有问题：
```js
// 有问题的写法：输出 3 个 3
for (var i = 1; i <= 3; i++) {
  setTimeout(() => console.log(i), 1000);
}
```
> - var 声明的变量没有块级作用域，i 是全局 / 函数级的变量，整个循环里只有一个 i；
> - setTimeout 是异步执行的，会等 1 秒后才执行回调函数；
> - 循环会在瞬间执行完（远小于 1 秒），此时 i 已经变成了 4（循环结束条件是 i>3），所以 1 秒后 3 个回调函数访问的都是同一个 i，最终输出 3 个 3（因为循环结束时 i 先到 4，回调取到的是循环结束前最后一次有效的值 3）。


闭包:
```js
// 闭包保留每次循环的变量值
for (var i = 1; i <= 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 1000);
  })(i);
}
// 输出：1、2、3（而非3个3）
```
> - 立即执行函数（IIFE）：(function(j) {...})(i) 是 “定义后立刻执行” 的函数，每次循环都会执行一次这个函数；
> - 参数传值：执行时把当前循环的 i 值（1、2、3 依次）传入，赋值给函数的参数 j；
> - **闭包保留作用域**：setTimeout 的回调函数会 “记住” 它所在的函数作用域，也就是每次循环创建的独立作用域里的 j；
> - 独立的变量副本：因为每次循环都创建了一个新的函数作用域，每个作用域里的 j 都是独立的（分别是 1、2、3），所以 1 秒后回调函数访问的是各自的 j，最终输出 1、2、3。

##### 3.  防抖 / 节流

```js
function debounce(fn, delay) {
  let timer = null; // 闭包保存定时器ID
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

### 四、 注意事项
- 内存泄漏风险：闭包会保留外层变量，若闭包长期引用（如挂载在 window 上），变量无法被垃圾回收，可能导致内存占用过高；
> 解决：不再使用时，手动将闭包引用置为 null（如 counter = null）。
- 性能轻微损耗：闭包的作用域链查找比普通变量慢，高频执行的代码需谨慎使用。
- 嵌套函数仅在内部执行 → 只是普通嵌套，不算闭包（无跨作用域执行）。
