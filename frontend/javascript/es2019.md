## ES2019特性

#### 数组扩展：Array.prototype.flat () /flatMap ()

**1. flat ()：扁平化嵌套数组**
> 将嵌套的数组 “拍平”，可指定扁平化深度，默认深度为 1；若想无限深度，传 Infinity

```js
// 基础用法：默认深度 1
const arr1 = [1, 2, [3, 4]];
console.log(arr1.flat()); // [1,2,3,4]

// 指定深度 2
const arr2 = [1, 2, [3, [4, 5]]];
console.log(arr2.flat(2)); // [1,2,3,4,5]

// 无限深度
const arr3 = [1, [2, [3, [4]]]];
console.log(arr3.flat(Infinity)); // [1,2,3,4]

// 自动过滤空元素（实用！）
const arr4 = [1, , 2, [3, , 4]];
console.log(arr4.flat()); // [1,2,3,4]
```

**2.  flatMap ()：map + flat（深度 1）**
> 先对数组执行 map 操作，再对结果执行 flat(1)，比单独调用 map + flat 更高效
```js
// 示例1：拆分字符串并扁平化
const arr = ['hello world', 'foo bar'];
// 普通 map + flat
console.log(arr.map(str => str.split(' ')).flat()); // ['hello','world','foo','bar']
// flatMap 简化
console.log(arr.flatMap(str => str.split(' '))); // 同上，更高效

// 示例2：过滤+转换（避免生成空数组）
const nums = [1, 2, 3, 4];
// 保留偶数并乘以 2
console.log(nums.flatMap(num => num % 2 === 0 ? [num * 2] : [])); // [4,8]
```

#### 字符串扩展：String.prototype.trimStart () /trimEnd ()（精准去除空格）

**替代 ES5 的 trimLeft()/trimRight()（非标准），精准去除字符串开头 / 结尾的空格（包括空格、制表符 \t、换行符 \n 等）；trim() 仍保留，用于去除两端空格**

```js
const str = '  hello world  ';

// 去除开头空格
console.log(str.trimStart()); // 'hello world  '
// 去除结尾空格
console.log(str.trimEnd()); // '  hello world'
// 保留原有 trim()（两端）
console.log(str.trim()); // 'hello world'
```

<br/>

#### 对象扩展：Object.fromEntries ()（反转 Object.entries ()）

**将「键值对数组」转换为对象，是 Object.entries() 的逆操作，解决了 “从数组快速构建对象” 的痛点**
```js
// 基础用法：数组转对象
const entries = [['name', '小明'], ['age', 18]];
const obj = Object.fromEntries(entries);
console.log(obj); // { name: '小明', age: 18 }

// 对比 Object.entries()（对象转数组）
const originalObj = { a: 1, b: 2 };
const entries2 = Object.entries(originalObj);
console.log(entries2); // [['a',1], ['b',2]]
console.log(Object.fromEntries(entries2)); // 还原为 originalObj
```

#### 可选的 catch 绑定（省略 catch 参数）

**当不需要使用 catch 块的错误参数时，可直接省略（无需写 catch (e) 或 catch (_)），简化代码**

```js
// ES2019 之前：必须写参数（即使不用）
try {
  JSON.parse('invalid json');
} catch (e) { // 即使不用 e，也必须声明
  console.log('解析失败');
}

// ES2019：省略 catch 参数
try {
  JSON.parse('invalid json');
} catch { // 无参数，更简洁
  console.log('解析失败');
}
```

#### Array.sort() : 稳定排序

**ES2019 规范要求 Array.sort() 必须是稳定排序（即排序后，相等元素的相对位置保持不变）。**
```js
// 稳定排序示例：按 age 排序，age 相同的元素保留原顺序
const arr = [
  { name: '小明', age: 18 },
  { name: '小红', age: 20 },
  { name: '小刚', age: 18 },
];
// 按 age 升序排序，小明应在小刚前面（原顺序）
arr.sort((a, b) => a.age - b.age);
console.log(arr.map(item => item.name)); // ['小明','小刚','小红']（稳定）
```

#### toString () 精准化

**ES2019 要求 toString() 返回函数的原始代码（包括空格、注释、换行），而非简化版本**

```js
// 示例
function fn() {
  // 这是注释
  return 1;
}
// ES2019 之前：可能返回 "function fn() { return 1; }"（丢失注释）
// ES2019 之后：返回原始代码（含注释、换行）
console.log(fn.toString());
/* 输出：
function fn() {
  // 这是注释
  return 1;
}
*/
```
