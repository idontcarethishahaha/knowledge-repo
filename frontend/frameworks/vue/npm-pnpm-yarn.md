## npm/yarn/pnpm

早期 npm
结构：递归处理嵌套，串行下载
问题：重复安装，速度极慢，占用空间大

npm@3+ / Yarn
结构：扁平化处理，并行下载
优点：目录浅，比早期npm快
缺点：幽灵依赖、多项目之间依旧有依赖重复安装，速度一般

pnpm
结构：全局存储 + hardlink硬指针 + 非扁平
优点：
- 无幽灵依赖
- 无重复安装
- 极省磁盘
- 极快
缺点：
- 旧库 / 工具不支持 symlink(符号链接)
- 很多旧项目大量依赖幽灵依赖，迁移到 pnpm 时会大面积报错，要逐个补依赖声明
